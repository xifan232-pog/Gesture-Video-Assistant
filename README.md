## 2. 💡 项目背景与痛点解决

1. **释放双手，提升生活体验**：
   在吃零食、做家务、躺靠休息或手上有油渍/脏污时，频繁操作鼠标或键盘极其不便。手势识别能实现“隔空控制”。
2. **跨窗口操控与焦点丢失**：
   主流网页端视频平台（抖音、B站等）虽支持键盘快捷键，但要求网页必须获得焦点。本项目通过系统级模拟事件与自动聚焦机制，解决了“跨窗口操控”与“焦点丢失”的痛点。
3. **轻量化桌面挂件设计**：
   悬浮窗占用空间小、支持无边框拖拽与折叠，在提供交互反馈的同时不会遮挡网页内容。

### 手势算法与映射逻辑

利用 MediaPipe Hands 提取手部 21 个 3D 骨骼关键点（Index: `0`-Wrist, `4`-Thumb_Tip, `8`-Index_Tip 等）：

| **手势动作**     | **对应功能** | **算法实现原理**                                             |
| ---------------- | ------------ | ------------------------------------------------------------ |
| 👌 **食拇指捏合** | 视频点赞     | 食指尖 (`Point 8`) 与大拇指尖 (`Point 4`) 的欧式距离 $d < 0.05$ |
| 👆 **食指朝上**   | 切上一个视频 | 食指尖 $Y_8 < Y_6$（高于关节），且 $Y_{12}, Y_{16}, Y_{20} > Y_{Knuckles}$（其余手指折叠） |
| 👇 **食指朝下**   | 切下一个视频 | 食指尖 $Y_8 > Y_6$（低于关节），且其余手指收起               |
| 👍 **大拇指朝上** | 系统音量增加 | 握拳状态下，大拇指尖 $Y_4 < Y_2$，且其余四指均处于收起/折叠状态 |
| 👎 **大拇指朝下** | 系统音量减小 | 握拳状态下，大拇指尖 $Y_4 > Y_2$，且其余四指均处于收起/折叠状态 |

## 6. 🐛 遇到的关键问题与解决方案

### 问题 1：手势误触与连续爆破触发

- **现象**：当手势停留在空中时，系统会在极短时间内反复触发“切视频”或“点赞”，导致页面疯狂翻页。
- **解决**：在后台识别线程 `GestureThread` 中引入 **时间冷却锁 (cooldown = 1.2s)** 机制。每次成功触发动作后更新 `last_action_time`，冷却时间内屏蔽后续手势识别。

### 问题 2：手势冲突（切视频与调音量互相误触）

- **现象**：最初设计使用“双手指”调音量，但由于 MediaPipe 识别手指弯曲时坐标存在边缘模糊，导致“伸单手指切视频”频繁被误判为“伸双手指调音量”。
- **解决**：重新设计手势逻辑，使用“食指”控制视频切换，“大拇指（点赞/踩手势）”控制音量。食指与大拇指各自独立检测，彻底消除了手势识别的几何冲突。

### 问题 3：UI 输入框抢占焦点，导致网页接收不到快捷键

- **现象**：在悬浮窗输入框粘贴网址后，键盘焦点停留在 UI 上，导致后台发送的 Up / Down / Space 键被 UI 截获，网页视频无法响应。
- **解决**：
  1. 在点击“跳转”后调用 `url_input.clearFocus()` 和 `self.clearFocus()` 彻底释放 UI 焦点。
  2. 结合 `QTimer.singleShot` 设置延时，等待浏览器加载完毕后，使用 `SystemController.play_video()` 自动点击网页视频左上区域（35% 相对坐标处），强行把焦点补回给浏览器。

### 问题 4：不同视频平台的点赞快捷键不统一/输入法干扰

- **现象**：有些平台使用 Z / L 键点赞，但经常受输入法中文模式影响导致按键失效。
- **解决**：将点赞机制优化为连续双击空格键模式 (`double space`)，回避了字母按键对输入法的依赖，实现了通用性强、响应迅速的点赞效果。
