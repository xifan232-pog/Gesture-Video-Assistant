# 🎬 刷视频手势助手 (Gesture Video Assistant)

基于 Python、PySide6 与 MediaPipe 开发的桌面悬浮手势交互小工具。无需触碰鼠标和键盘，仅通过摄像头识别手势即可轻松刷抖音、B站等视频平台。

##  1. 项目介绍 

**刷视频手势助手**是一款专注于提升短视频/长视频浏览体验的辅助工具。它会在桌面生成一个透明、置顶的微型悬浮窗，通过摄像头实时捕捉并分析用户的手势，将其转化为网页端的控制指令。

###  核心功能一览

- **视频切换**：伸出食指向上/向下切上/下一个视频。
- **快捷点赞**：食指与大拇指捏合，快速触发视频点赞。
- **音量调节**：大拇指朝上/朝下直接控制系统全局音量。
- **自动聚焦与播放**：输入网址跳转后，自动聚焦网页视频区域并触发播放。
- **数据统计与预览**：实时预览摄像头画面，并自动统计已看视频数量。

## 2. 项目背景

1. **释放双手，提升生活体验**：

   在吃零食、做家务、躺靠休息或手上有油渍/脏污时，频繁操作鼠标或键盘极其不便。手势识别能实现“隔空控制”。

2. **多终端兼容与便捷操作**：

   主流网页端视频平台（抖音、B站等）虽支持键盘快捷键，但要求**网页必须获得焦点**。本项目解决了“跨窗口操控”与“焦点丢失”的痛点。

3. **轻量化桌面挂件设计**：

   悬浮窗占用空间小、支持无边框拖拽与折叠，在提供交互反馈的同时不会遮挡网页内容。

##  3. 项目架构

本项目采用了典型的 **MVC/三层架构设计**，模块清晰、高内聚低耦合：

Plaintext

```
       ┌────────────────┐
       │   ges_ui.py    │ (视图层：PySide6 悬浮窗)
       └───────┬────────┘
               │ Signal / Slot
       ┌───────▼────────┐
       │ ges_thread.py  │ (逻辑层：OpenCV + MediaPipe 多线程识别)
       └───────┬────────┘
               │ 静态方法调用
       ┌───────▼────────┐
       │ controller.py  │ (控制层：PyAutoGUI 模拟系统硬件事件)
       └────────────────┘
```

### 技术栈说明

- **界面框架 (`PySide6`)**：
  - 使用 `Qt.WindowStaysOnTopHint | Qt.FramelessWindowHint | Qt.Tool` 构建无边框、顶层悬浮窗。
  - 基于 `QThread` 实现多线程，将耗时的图像采集与 AI 推理放到后台，保障 UI 界面流畅不卡顿。
- **计算机视觉与手势识别 (`OpenCV` + `MediaPipe`)**：
  - **OpenCV**：负责捕获摄像头视频帧、图像镜像翻转及格式转换。
  - **MediaPipe Hands**：轻量级手部 21 个 3D 骨骼关键点检测模型，用于提取手指坐标。
- **系统自动化交互 (`PyAutoGUI` & `webbrowser`)**：
  - 将识别到的手势转换为 `PyAutoGUI` 的键盘/鼠标事件（如 `press('up')`、`press('space')`、`press('volumeup')`）。

##  4. 遇到的问题及解决方案

在开发过程中，针对手势识别的灵敏度、UI 抢占焦点以及手势冲突等问题进行了多次迭代：

### 问题 1：手势误触与连续爆破触发

- **现象**：当手势停留在空中时，系统会在极短时间内反复触发“切视频”或“点赞”，导致页面疯狂翻页。
- **解决方案**：在后台识别线程 `GestureThread` 中引入 **时间冷却锁 (`cooldown = 1.2s`)** 机制。每次成功触发动作后更新 `last_action_time`，冷却时间内屏蔽后续手势识别。

### 问题 2：手势冲突（切视频与调音量互相误触）

- **现象**：最初设计使用“双手指”调音量，但由于 MediaPipe 识别手指弯曲时坐标存在边缘模糊，导致“伸单手指切视频”频繁被误判为“伸双手指调音量”。
- **解决方案**：重新设计手势逻辑，使用“食指”控制视频切换，**“大拇指（点赞/踩手势）”控制音量**。食指与大拇指各自独立检测，彻底消除了手势识别的几何冲突。

###  问题 3：UI 输入框抢占焦点，导致网页接收不到快捷键

- **现象**：在悬浮窗输入框粘贴网址后，键盘焦点停留在 UI 上，导致后台发送的 `Up` / `Down` / `Space` 键被 UI 截获，网页视频无法响应。
- **解决方案**：
  1. 在点击“跳转”后调用 `url_input.clearFocus()` 和 `self.clearFocus()` 彻底释放 UI 焦点。
  2. 结合 `QTimer.singleShot` 设置延时，等待浏览器加载完毕后，使用 `SystemController.play_video()` 自动点击网页视频左上区域（`35%` 相对坐标处），强行把焦点补回给浏览器。

###  问题 4：不同视频平台的点赞快捷键不统一/输入法干扰

- **现象**：有些平台使用 `Z` / `L` 键点赞，但经常受输入法中文模式影响导致按键失效。
- **解决方案**：将点赞机制优化为**连续双击空格键模式 (`double space`)**，回避了字母按键对输入法的依赖，实现了通用性强、响应迅速的点赞效果。
